Receipt parsing API requirements:
Create app/api/bills/parse/route.ts.

The route must:

1. Accept multipart/form-data.
2. Read an image field named image.
3. Validate MIME type:

   * image/jpeg
   * image/png
   * image/webp
4. Reject files larger than 10MB.
5. Convert uploaded file to Buffer.
6. Preprocess image using sharp:

   * rotate based on metadata
   * resize to max width 1600px
   * output jpeg or webp
7. Send the processed image to Gemini API.
8. Ask Gemini to return structured JSON only.
9. Validate Gemini response using Zod.
10. Normalize money fields to numbers.
11. Generate stable IDs for bill items and charges.
12. Store parsed bill in SQLite.
13. Return billId and normalized bill JSON.

Gemini extraction schema:
The parsed bill should contain:

* merchantName: string | null
* billDate: string | null
* currency: string, default VND if unclear
* items:

  * id: string
  * name: string
  * quantity: number
  * unitPrice: number
  * totalPrice: number
  * confidence: number
* charges:

  * id: string
  * name: string
  * type: "tax" | "service" | "tip" | "other"
  * amount: number
* discount: number
* subtotal: number
* total: number
* rawText: string
* warnings: string[]

Split engine requirements:
Create pure TypeScript logic in src/lib/split/splitEngine.ts and src/lib/split/settlement.ts.

Rules:

1. Each item can be assigned to one or more participants.
2. Item totalPrice is split equally among assigned participants.
3. Tax, service charge, tip, and other charges default to proportional allocation based on each participant's item subtotal.
4. Discount defaults to proportional allocation based on each participant's item subtotal.
5. Support equal allocation for charges and discount.
6. Support rounding:

   * none
   * nearest_100
   * nearest_500
   * nearest_1000
7. If one person paid the whole bill, settlement should show who transfers how much to that payer.
8. Throw validation error if any item has no assigned participant.
9. Throw validation error if participants list is empty.
10. Throw validation error if assignment references unknown item or participant.
11. Keep calculation deterministic and unit-testable.

Bill validation requirements:
Create src/lib/bill/validateBill.ts.

Validation:

* calculatedSubtotal = sum(items.totalPrice)
* calculatedTotal = calculatedSubtotal + sum(charges.amount) - discount
* Compare calculatedTotal with bill.total.
* Allow a small tolerance for rounding.
* If mismatch exists, add warning:
  "Parsed total does not match calculated total. Please review the bill."
