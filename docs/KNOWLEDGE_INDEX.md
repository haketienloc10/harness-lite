# Knowledge Index — repo-harness

> "Accessed knowledge": the onboarding map agents read before changing code.
> Generated by `harness-cli knowledge`. Top-Level Structure and Key Technologies
> are regenerated each run; Purpose and Key Concepts are authored and preserved
> between the markers.

## Purpose

<!-- KNOWLEDGE:PURPOSE:BEGIN -->

A repository-level operating harness that turns any software repo into an
agent-ready workspace. It gives coding agents the project context they need
_before_ changing code — where to start, the product contract, how risky the
work is, what proof is required, and which decisions to inherit — through a
Docs-First 7-stage workflow and a durable SQLite layer driven by `harness-cli`.

<!-- KNOWLEDGE:PURPOSE:END -->

## Top-Level Structure

- `AGENTS.md` — Entrypoint and onboarding for AI agents; points into
  `_harness/`.
- `Cargo.lock` — Pinned dependency versions for the Cargo workspace.
- `Cargo.toml` — Cargo workspace manifest (member: `crates/harness-cli`).
- `README.md` — Human onboarding: what Harness is and why it exists.
- `_harness/` — Core agent execution framework: workflow, standards, CLI
  reference, skills.
- `crates/` — Rust source for the `harness-cli` durable-layer CLI.
- `docs/` — Human reference: architecture, glossary, decisions, stories,
  templates.
- `guides/` — Long-form harness guides and worked process examples.
- `install-harness-cli.sh` — Installs the prebuilt `harness-cli` binary into a
  repo.
- `install.sh` — Installs the harness payload (`_harness`, `docs`, `scripts`,
  configs) into a target repo.
- `scripts/` — Harness automation: prebuilt CLI binary (`bin/`) and SQLite
  schema (`schema/`).

## Key Concepts

<!-- KNOWLEDGE:CONCEPTS:BEGIN -->

See `docs/GLOSSARY.md` for full definitions. Core terms:

- **Harness Workflow** — 7-stage gated process from intake to growth.
- **Lane** — risk-based context budget: tiny, normal, or high-risk.
- **Hard Gate** — high-risk trigger requiring human review or a decision record.
- **Story Packet** — story-sized doc set (overview, execplan, design,
  validation).
- **Durable Layer** — operational state (intake, story, decision, trace) kept in
  `harness.db` via `harness-cli`.
- **Product Delta vs Harness Delta** — changes to the app vs changes to the
  harness itself.
- **Trace / Friction** — telemetry of agent actions and logged obstacles.
- **Skill** — JIT-loaded procedure with a mechanical `VERIFY` + `EXIT`.
- **Quality Gate** — three-lens review: architecture, security, maintainability.
- **Knowledge Index** — this file; the "Accessed knowledge" onboarding map,
generated by `harness-cli knowledge`.
